## 2. What's wrong
1. **`tsconfig.base.json` and `eslint.config.mjs` currently live at repo root.** Spec requires them in `packages/config/`, which doesn't exist yet as a package. Every prompt from Wave 1 onward assumes `packages/config` exists with these files — leaving them at root breaks that assumption for every later pebble.
2. **`tsconfig.base.json`'s path alias `"@vergeo/ui": ["packages/ui/src/index.ts"]` is a single-file barrel alias.** This directly conflicts with the binding convention in `docs/plan/03-waves.md` §Shared-file conventions #1: **"No barrel files in `packages/ui` — deep imports only."** Multiple parallel M02 component pebbles (Wave 2) will each own their own file under `packages/ui/src/` — if they all have to export through one `src/index.ts`, that's exactly the shared-file contention the wave plan is designed to avoid.
3. **Accepted, no action needed:** `.env.example`, `.node-version`, `.prettierrc`, `.prettierignore`, `commitlint.config.cjs`, `lefthook.yml`, `scripts/verify-scaffold.mjs` were added beyond the pebble's file list. These are useful and the founder has accepted them as a disclosed scope addition — **do not remove them**. Just fold the prettier config into the relocation below (see step 3).
