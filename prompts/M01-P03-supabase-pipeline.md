## 4. Implementation spec
- **`0001_extensions.sql`:** `create extension if not exists` for **`pgcrypto`, `pg_trgm`, `vector`** (pgvector). Nothing else — domain tables are M03's.
- **Migration convention (document in the workflow doc):** one migration file per pebble, `NNNN_slug.sql`, numbers assigned in each Phase 3 prompt; **additive-only after M03 merges**.
- **`scripts/gen-types.sh`:** runs `supabase gen types typescript --local` (with a `--project-id` variant documented for staging), writes `packages/types/src/db.ts`, fails loudly if the CLI or stack is unavailable. Committed types = source of truth; CI (P06) will fail on drift.
- **`packages/types`:** minimal package (name `@vergeo/types`, deep-import consumable, no barrel beyond `src/db.ts` itself).
- **`docs/ops/supabase-workflow.md`:** local stack up/reset (`supabase start`, `supabase db reset`), diff → new migration, push to staging/prod projects, typegen step, env/secret names (**names only, never values**), and the additive-only rule.
