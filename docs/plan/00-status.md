### ⚠ Wave-6 integration debt (CI-invisible — each pebble tests in isolation)

Two real runtime breaks in the **onboarding → KYC** flow (not blocking: the flow isn't reachable end-to-end until M13-P02 KYC-approval UI + M12-P03 listing-create land):

1. **KYC endpoint contract mismatch.** UI (`onboarding/_lib/kyc-client.ts`) calls `GET /kyc/application`, `PATCH /kyc/application/draft`, `POST /kyc/application/{submit,resubmit}`; backend (M12-P02 `kyc.py`) actually serves `GET /kyc/status`, `POST /kyc/{submit,resubmit}` with different request/response shapes (`{tier, doc_storage_paths[], momo_phone, momo_operator?, legal_name}` ↔ backend `KycStatusResponse`). Backend is authoritative (tested + DB writes). Reconciliation = align the UI client to the backend contract + map the data model (incl. collecting `legal_name` for the momo name-match; UI currently collects only business name/category).
2. **No private-doc signing endpoint exists.** UI expects `POST /media/sign {resource_kind:'kyc_doc'}` → a Supabase Storage **private-bucket** signed upload (`{bucket, path, token, signed_url}`). But `/media/sign` is **Cloudinary-only, hard-locked to `Literal["listing"]`** (public media) — a different system. KYC docs need a Supabase-Storage private signed-upload endpoint that **no pebble built**. This is missing scope, not just a mismatch.

**Resolution — `prompts/M12-P02b-kyc-integration-reconciliation.md` (ready).** On close tracing, this is **pebble-sized, not a quick convergence edit**: it needs a private `kyc-docs` Storage bucket declared in `config.toml` (which changes what `supabase db start` does in the `db`+`rls` CI jobs — **untestable locally without Docker**), a signed-upload endpoint, the client repointed to `/kyc/status|submit|resubmit` with the real body, and a new `legal_name` field for the name-match (backend also has **no `resubmit` status** [uses `rejected`], **no draft endpoint** [localStorage only], and stores docs as a flat `doc_storage_paths[]`). Hand-editing the storage/`config.toml` piece onto green master and gambling on the untestable `db start` change — for a PII endpoint — is the wrong risk. So it's routed through a **CI-validated PR branch** (the restored CI exercises `db start` with the bucket) rather than a direct-to-master convergence commit. Grounded with exact contract shapes in the prompt.
