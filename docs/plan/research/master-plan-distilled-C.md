## 2. PRODUCT/BUSINESS
- Identity: "Convergeo is not a marketplace. It is a **commerce-powered discovery platform**" — "Discover Everything Zambia Has to Offer"; "hub of logos" vision (business directory is a first-class feature).
- Key competitive insight (Q71): biggest competitor is **WhatsApp informal sellers**, not Jumia. Moat = discovery + trust (escrow) + convenience (delivery/pickup) + confidence (reviews).
- Multivendor platform for products + services + events across all 10 Zambian provinces; buyers, sellers, service providers, event organisers. B2B (bulk/RFQ) + B2C.
- Solo founder (Prosper Kaluba) + AI tools; zero headcount; company = private limited registered with PACRA ("Vergeo Technologies Ltd., registered in Zambia", p7).
- Supply-first strategy: onboard vendors first, free listings 3-6 months (Q3). Quiet launch: invite-only beta → public (Q54).
- Revenue architecture (p16-17), 4 layers:
  1. **Commissions, variable by category**: 5% electronics, 8% home goods, 10% fashion/beauty, 12% services, 5% event tickets.
  2. **Vendor subscriptions**: Free (20 products, basic profile/analytics, standard commission); Bronze K99/mo (100 products, verified badge, featured-in-category, −1% commission); Silver K249/mo (unlimited products, priority search placement, rich profile, advanced analytics, bulk upload); Gold K499/mo (all + homepage rotation, dedicated support, API integration, competitor pricing insights, video profile).
  3. **Events**: 5% ticket commission + paid promoted event listings.
  4. **Promoted listings** (future, auction ad slots — not at launch).
- Vendor payment terms: no monthly fees required — "only pay when you sell" (p7); payouts within 48 hours.
- Expansion: Zimbabwe first (Q73); seek investment after 1,000+ orders/month traction (Q74); 5-year super-app vision (commerce + payments + logistics + services + events, Q75).
- Deck market claims (aspirational UI numbers, p1-2): 12,400+ products, 840+ vendors, 8 product categories, 6 service categories, 24 events/month, 12,000+ event attendees, 10 provinces; 78% of Zambian internet users are mobile.

## 3. FEATURES & MODULES
**Customer:** product browse/search (Meilisearch, typo-tolerant, faceted filters: category/price/vendor/rating/availability/location); autocomplete + later pgvector semantic search; homepage (hero search → events → categories → trending → vendor spotlight; mini expandable footer); product detail with image gallery, variants (size/color), multi-vendor comparison view (price/rating/delivery time, sort by nearest); cart (anonymous + user, Redis-persisted, merge on login); multi-step checkout (address/pickup → delivery options → payment → confirm); order tracking (status timeline); reviews (verified-purchase only, 1-5 stars, multi-dimensional quality/delivery/communication, photo/video, vendor responses, moderation flags); QR + PIN pickup verification; referral program (K20/K20); PWA installable with offline catalog browsing; i18n EN + Bemba + Nyanja; auth via phone OTP (Africa's Talking) or email + Google/Facebook social login; location tags on listings: Nearest / Cheapest / International / 7-Day Wait.
**Vendor:** tiered onboarding (instant basic; 1-3 days verified) with tiered KYC (NRC photo individuals; PACRA docs companies); status machine Applied→Under Review→Approved/Rejected; storefront profile (location, hours, description; richer per tier — video, team pages, certifications, social links); product CRUD (up to 8 images, drag-reorder, draft→pending→published approval flow); inventory manual + CSV bulk + API/POS; dashboard analytics (progressive by tier: sales/orders free; conversion, traffic sources, competitor insights paid); order management (confirm/reject with reason codes, ship, track, pick/pack/ship workflow); QR scanner (camera); CSV bulk upload; B2B tiered volume pricing + RFQ (later).
**Admin:** vendor approval queue + KYC review, product moderation, user management (suspend/ban), dispute management (evidence review, full/partial refunds), platform dashboard (GMV, orders, users, vendors, charts), commission engine per-vendor/per-category, payout tracking/reconciliation, CSV/Excel exports, role-based access (superadmin/moderator), audit log.
**Platform modules:** escrow ledger system; order state machine (Placed→Confirmed→Processing→Shipped→Delivered→Completed, audited); dispute resolution (auto escrow release after ~7-day post-delivery window); notifications (email SendGrid/Mailgun, SMS, in-app real-time, WhatsApp with SMS fallback, per-user channel preferences); events & ticketing (dynamic QR refreshing every 60s to block screenshot forwarding); business directory (separate nav tab); quote-request services marketplace ("post a job, get quotes" — no booking calendar); city guides (Lusaka, Livingstone, Ndola/Copperbelt) + later AI trip planner; in-platform pre-purchase messaging (contacts shareable post-purchase only); AI features roadmap: AI product descriptions from photo → fraud detection → price optimization; personalized recommendations; conversational search ("find me a red dress under K200 near me"); abandoned cart recovery (n8n + SMS/WhatsApp); promoted listings; Phase-4: airtime/bill payments, Convergeo Pay wallet (credits, P2P), vendor financing (MFI, sales-data based), AR try-on, voice search in Bemba/Nyanja, React Native Android app, pickup point network (Shoprite, Puma stations).

## 5. PAYMENTS / LOGISTICS / COMPLIANCE
**Payments:** DPO Pay primary (mobile money + Visa/Mastercard); Lenco secondary (bank transfers + developer API; Phase 2 redundancy); mobile money non-negotiable day one — MTN MoMo + Airtel Money (+ Zamtel Kwacha in roadmap/deck); 70%+ of Zambian digital payments are mobile money; payment abstraction layer (strategy pattern); webhooks with idempotency + signature verification; transaction ledger; USSD prompt flow UI; full escrow (funds held until buyer confirms; #1 differentiator); escrow release = 48h after delivery confirmation OR 7 days after shipping, whichever first; ledger-based, not bank sub-accounts; vendor settlement 48h at launch ("fastest in Zambia" vs competitors' 7-14 days), instant settlement only after K10,000+ float (~month 3-4); COD for orders under K500 + pay-at-pickup via mobile money for larger; display any currency, settle in ZMW; manual bank-transfer fallback as emergency; long-term: partner/acquire licensed fintech, don't build one.
**Logistics:** Yango API (urban) + local couriers (Zampost, private) behind a delivery aggregation layer; vendors list nationwide but **delivery Lusaka-only first 60 days**, pickup everywhere; Copperbelt (Ndola, Kitwe) delivery from ~month 3; 1-3 day standard, no same-day (express later); hybrid fulfillment — platform-managed for small items, vendor-managed for large (furniture/appliances); pass-through partner tracking (webhooks, no GPS build); delivery zones + fee calc by zone/vendor location, geocoding; pickup point model (location/hours/contact); QR (unique per order) + PIN backup verification, scan triggers escrow release; free delivery above K200 order value (Q6); no returns in MVP — refund-only for faulty/wrong items.
**Compliance/tax:** PACRA company registration (week 1, parallel with dev); consult tax advisor; below K800K turnover = no mandatory VAT registration, but VAT-ready architecture day one (VAT toggle per category); GDPR-*ready* architecture (encryption, consent tracking, audit logs) but formally comply only with **Zambia Data Protection Act** at first (full GDPR legal setup K25,000-K50,000+ deferred); privacy policy + ToS (Zambia DPA-compliant) in week 7; dispute escalation: automated first, human review for K500+; ZRA not mentioned by name.
**WhatsApp:** competitor (informal sellers, Q71); support channel (WhatsApp + AI chatbot + phone for high-value, Q64); vendor beta support WhatsApp group; launch promoted via WhatsApp groups; WhatsApp Business API (Meta Cloud API or BSP) — approved templates for order confirmation/shipping/delivery, customer + vendor alerts, opt-in/opt-out, delivery-receipt/reply webhooks, SMS fallback; abandoned-cart nudges via WhatsApp; Phase 3 (master plan): order placement + tracking via WhatsApp.
